回顧本次完整對話，識別並保存有長期價值的模式。

## 核心原則

1. **更新優先於新建**：首要任務是找到可以更新的既有記錄，而非建立新的
2. **精簡克制**：每次最多 1-2 項變更（更新 + 新建合計），寧可跳過也不要建立低價值記錄
3. **正確區分層級**：全域 vs 專案

## Phase 1：掃描對話

回顧對話，尋找以下候選模式（沒有就直接報告「本次無需保存」並結束）：

| 類型 | 特徵 | 只記錄如果... |
|------|------|-------------|
| **Feedback** | 使用者糾正做法（「不要…」「改用…」「以後…」） | 適用於未來對話，不是一次性指令 |
| **Skill** | 執行了 3+ 步驟的多步驟工作流程 | 未來可能重複使用 |
| **Project** | 專案資訊（決策、截止日期、架構） | 無法從程式碼或 git 推導 |
| **User** | 使用者角色、專業、偏好 | 之前未記錄過 |

## Phase 2：層級判斷

對每個候選模式：

**全域層級**（存在 `~/.claude/` 下）：
- 適用於所有專案的通用規則或偏好 → 更新 `~/.claude/CLAUDE.md` 對應章節
- 跨專案可重複使用的工作流程 → `~/.claude/commands/*.md`
- 使用者個人資訊 → auto memory 系統（user 類型）

**專案層級**（存在當前專案目錄下）：
- 涉及特定技術棧、檔案路徑、架構 → 更新專案 `CLAUDE.md`
- 專案專屬的工作流程 → `.claude/commands/*.md`
- 專案脈絡（決策、團隊、時程） → auto memory 系統（project 類型）
- 行為修正涉及此專案特定技術 → auto memory 系統（feedback 類型）

## Phase 3：去重與合併（最關鍵 — 必須在保存前完成）

依序檢查：

1. 讀取 `~/.claude/CLAUDE.md`
2. 讀取當前專案 `CLAUDE.md`
3. 讀取 auto memory 系統的 `MEMORY.md` 索引，以及標題相關的記憶檔案
4. 列出 `~/.claude/commands/*.md`（全域 skill）
5. 列出 `.claude/commands/*.md`（專案 skill）

對每個候選模式做決策：

| 情況 | 動作 |
|------|------|
| 既有記錄已完整涵蓋此內容 | **跳過** |
| 既有記錄部分涵蓋 → 有新資訊可補充 | **更新**該記錄 |
| 全新主題且確有長期價值 | **新建** |
| 價值不確定或只出現過一次 | **跳過**，等未來再次出現時再保存 |

## Phase 4：執行

**更新既有記錄：**
- CLAUDE.md 規則：找到對應章節，補充新條目或精煉既有描述
- Memory 檔案：讀取原檔 → 合併新資訊 → 保持 frontmatter 格式
- Skill 命令檔：讀取原檔 → 補充新步驟或改進既有步驟

**新建記錄（僅確認無重複時）：**
- Memory：使用 auto memory 格式（frontmatter + 內容），更新 MEMORY.md 索引
- Skill：建立 `.md` 命令檔，含 `description` frontmatter 和可執行步驟
- CLAUDE.md：在對應章節下加入條目
